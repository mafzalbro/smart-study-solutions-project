1. Recommend a Resource

Endpoint: GET /resources/recommend

Body: None

2. Get a Resource by ID

Endpoint: GET /resources/:id

Example URL: GET /resources/60f7187b9f1b2a456c6d71f1

Body: None

3. Add a New Resource

Endpoint: POST /resources

Body (JSON):

{ "title": "Sample Resource Title", "author": "Sample Author", "genre": "Sample Genre", "description": "This is a sample resource description.", "status": true, "slug": "sample-resource-title", "type": "book", "ai_approval": true, "tags": ["sample", "resource", "test"] }

4. Update a Resource by ID

Endpoint: PUT /resources/:id

Example URL: PUT /resources/60f7187b9f1b2a456c6d71f1

Body (JSON):

{ "title": "Updated Sample Resource Title", "author": "Updated Sample Author", "genre": "Updated Sample Genre", "description": "This is an updated sample resource description.", "status": true, "slug": "updated-sample-resource-title", "type": "book", "ai_approval": false, "tags": ["updated", "sample", "resource"], "rating": 4.5, "ratingCount": 10, "likes": 15, "dislikes": 2 }

5. Delete a Resource by ID

Endpoint: DELETE /resources/:id

Example URL: DELETE /resources/60f7187b9f1b2a456c6d71f1

Body: None

6. Get All Resources with Optional Sorting, Filtering, and Searching

Endpoint: GET /resources

Example URL: GET /resources?page=1&limit=5&sortBy=title:asc&filterBy={"status":true}&query=sample

Body: None

7. Like a Resource

Endpoint: POST /resources/:id/like

Example URL: POST /resources/60f7187b9f1b2a456c6d71f1/like

Body: None

8. Dislike a Resource

Endpoint: POST /resources/:id/dislike

Example URL: POST /resources/60f7187b9f1b2a456c6d71f1/dislike

Body: None

9. Rate a Resource

Endpoint: POST /resources/:id/rate

Example URL: POST /resources/60f7187b9f1b2a456c6d71f1/rate

Body (JSON):

j
{ "rating": 5 }

Summary

To test these endpoints in Postman, follow these steps:

  1. Create a new request for each endpoint.
  2. Replace :id in the URL with the actual resource _id.
  3. Set the HTTP method (GET, POST, PUT, DELETE) as specified.
  4. For requests requiring a body (POST, PUT), select the Body tab in Postman, choose raw, and set the body type to JSON.
  5. Copy and paste the example JSON body into the Body tab.

By following these steps, you should be able to test each of the resource management endpoints effectively.
